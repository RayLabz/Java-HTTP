Java HTTP

A Java utility library that makes HTTP requests easier to work with.
Download

The Java HTTP library simplifies the execution of HTTP requests in Java through standardization. It allows you to quickly build a request using a builder pattern, add parameters, execute the request and receive a response and its data. While standardizing the aspects of HTTP requests, the library still maintains some versatility, allowing you to a) define request headers and read response headers and b) define request timeouts.

The package requires Java 8+

Guide

Download and import

You can automatically import Java-HTTP in your project using Maven or Gradle:

Maven:

<dependency>
  <groupId>com.raylabz</groupId>
  <artifactId>responz</artifactId>
  <version>1.0.2</version>
</dependency>

Gradle:

implementation 'com.raylabz:responz:1.0.2'

Alternatively, you can download Java-HTTP as a .jar library:

Download .jar
Making an HTTP GET request

You can make a GET request, add parameters, execute it and receive the response by simply using: 

HTTPResponse response = new HTTPRequest.Builder("http://www.example.com/api/endpoint", RequestMethod.GET)
    .addParam("myParam", "myValue") //Add parameter(s)
    .build() //Build the request
    .call(); //Make the call
Getting the response

You can get the response data of a call by using the response object:

//Get the response data:
System.out.println("Response code: " + response.getStatus()); //Fetch the response code/status
System.out.println("Response content: " + response.getContent()); //Fetch the response content (body)
System.out.println("Response header field 'Content-Type': " + response.getHeaderField("Content-Type").get(0));
Making an HTTP POST request

Alternatively, you can also make and execute a POST request, simply by changing the RequestMethod: 

//Perform a POST HTTP request:
HTTPResponse response2 = new HTTPRequest.Builder("http://www.example.com/api/endpoint", RequestMethod.POST)
    .addParam("myParam", "myValue") //Add parameter(s)
    .build() //Build the request
    .call(); //Make the call
Defining request headers

You can define a header for your request using the setRequestProperty() method: 

HTTPResponse response = new HTTPRequest.Builder("http://www.example.com/api/endpoint", RequestMethod.GET)
    .addParam("myParam", "myValue") //Add parameter(s)
    .setRequestProperty("User-Agent", "Java")
    .build() //Build the request
    .call(); //Make the call
MIME types

You can quickly define a frequently used MIME type for your request using the setMimeType() method: 

HTTPResponse response = new HTTPRequest.Builder("http://www.example.com/api/endpoint", RequestMethod.GET)
    .addParam("myParam", "myValue") //Add parameter(s)
    .setMimeType(MimeType.CONTENT_TYPE_JSON)
    .build() //Build the request
    .call(); //Make the call

Alternatively, if the MIME type you want does not exist as a MimeType, you can use the setCustomMimeType() method to set the type as a string: 

.setCustomMimeType("text/*")
Timeouts

You can define the connection timeout and data read timeout limits using the setConnectTimeout() and setReadTimeout() methods: 

.setConnectTimeout(20000) //Set the connection timeout
.setReadTimeout(15000) //Set the data read timeout
Status checks

You can quickly check if the status is successful by using the isSuccess() method: 

if (response.isSuccess()) {
    //TODO - Success, do something with the response.
}
else {
    //TODO - Failure, show error message.
}

Alternatively, you can also get the detailed HTTP status of the response by using the getStatus() method: 

int status = response.getStatus();
